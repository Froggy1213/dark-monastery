## Architecture

**Dark Monastery** is a text-based Dark Fantasy RPG where the game master is Google Gemini. The player types actions in a terminal; Gemini generates the next story beat as structured JSON, which the engine deserializes back into game state.

### Package map

| Package | Purpose |
|---|---|
| `internal/game` | `GameState` struct and `NewPlayer()` factory — the canonical state shape serialized to/from Gemini |
| `internal/ai` | Gemini API client (`gemini-2.5-flash-lite`) — sends state + player action, returns updated `GameState` JSON |
| `cmd/server` | Terminal game loop — reads stdin, calls AI, prints state |
| `internal/handlers` | **Stub** — planned HTTP/WebSocket handlers for a web frontend |
| `internal/memory` | **Stub** — planned conversation history and vector DB for lore retrieval |
| `internal/storage` | **Stub** — planned in-memory and Postgres game session storage |
| `static/` | **Stub** — planned HTML/JS/CSS frontend |

### Data flow

```
Player input (stdin)
  → cmd/server/main.go (game loop)
    → ai.GenerateNextTurn(apiKey, *GameState, playerAction)
      → POST to Gemini API with system prompt + current state JSON
      ← Gemini returns JSON matching GameState struct
    ← *GameState updated
  → Print location, condition, inventory, message to terminal
```

### Key design decisions

- **Gemini is the game engine.** There is no rule-based game logic — all story progression, stat changes, inventory updates, and location transitions are generated by the LLM.
- **State is a flat struct.** `GameState` has no nested objects or relationships — it's a simple bag of fields designed to be easy for the LLM to produce as valid JSON.
- **The system prompt is in Russian** and lives inline in `internal/ai/gemini.go`. It instructs Gemini to act as a strict game master, always return valid JSON matching the `GameState` schema, and drive the plot forward with atmospheric dark fantasy narration.
- **Exponential backoff** in `gemini.go`: up to 3 retries on HTTP 429 (rate limit) and 503 (overload), with `attempt*2` second delays.
- **JSON mode** is enforced via Gemini's `responseMimeType: "application/json"` generation config.

### Project phase

The project is in **Phase 1** (terminal prototype). All packages except `internal/game`, `internal/ai`, and `cmd/server` contain only package declarations — they represent planned features for later phases (web frontend, persistent storage, conversation memory, vector search over lore).
